Clarification Questions & Answers

The clarification questions we sent, the client's answers, and what each means for the solution. (This is the answered record — not a new set of questions.)

Theme We asked Client answer What it means for us
Priority The one thing to get right first? A complete self-service request → approval flow, connected to the booking platform The MVP centres on this core journey
Goal Sell software, or grow the business? Grow and reduce manual work — not a product to sell Optimise for adoption and less agent effort
Roles Which types of user? Six, including a Key Travel Agent; staff must be able to book for others Roles and "book for others" from the MVP
Approvals How complex? 1–3 configurable levels; triggered by cost, destination, seniority, cabin class A flexible, step-by-step approval flow
Rule-breaking Block or allow? Left to us We propose warn + justify (a nudge, not a block)
Existing system Reuse or rebuild? Reuse the search/book/change/cancel functions; documentation is incomplete Build a safe link and investigate it early
Invoicing Is there an automatic link? No — finance runs on Microsoft Business Central The platform creates invoices and sends figures to Business Central
Payment How is it paid? Central invoice + company card No individual traveller payments in the MVP
Scale How many organisations? Pilot first → hundreds; NGO/humanitarian to start Built for many separate organisations, pilot-focused
Security Any requirements? Secure login (SSO/MFA) and data privacy (GDPR) Built in from day one
Constraints Time, budget, team? ~9 months, limited budget, little client tech capacity, launch fast A small MVP, built and run by Key Travel, with flexible scope
